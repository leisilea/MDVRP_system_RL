# MDVRP算法机制详解合集

本文档整合了遗传算法(GA)、粒子群算法(PSO)和蚁群算法(ACO)三类元启发式算法在MDVRP问题求解中的核心机制。

---

## 第一部分: GA+RL混合初始化策略

### 核心创新点

为提升遗传算法的初始种群质量,本研究创新性地提出了强化学习辅助初始化策略(GA+RL);该方法利用预训练强化学习模型(RouteFinder+RL4CO)生成高质量初始解;实验结果表明,该策略在中大规模问题实例上的求解效果优于传统随机初始化方法。

### RL混合初始化机制

**策略设计**:
- RL种子占种群的20%(20个个体)
- 每个depot采样20个解
- 其余80个个体采用随机初始化
- 平衡质量与多样性

**实验结果**:
- 小规模实例(≤100客户): 效果有限,主要因预训练模型缺乏小规模检查点
- 中大规模实例(100-240客户): 改进幅度有限,时间开销增加约20秒
- 大规模实例(≥240客户): 在P08-P11实例组中实现1.9%的性能提升

**收敛特征**:
- RL初始化在前100-200代内提供显著起步优势
- 高质量初始解提升后续迭代中的缓存命中率
- 大规模实例通常在800代左右完成收敛

## 第二部分: PSO算法机制详解

### 1. 多进程实现情况

**当前状态**: 未实现多进程,采用顺序执行方式

**执行流程**:
1. 仓库1 PSO优化 (100粒子 × 350代)
2. 仓库2 PSO优化 (100粒子 × 350代)
3. 仓库3 PSO优化 (100粒子 × 350代)
4. 仓库4 PSO优化 (100粒子 × 350代)
5. 跨仓库边界优化

**并行化潜力**:
- 4个仓库的优化过程完全独立
- 理论加速比: 接近4倍(取决于CPU核心数)
- 预期性能提升: P01从2.87秒降至0.8-1.0秒

### 2. 仓库分配后的客户交换机制

#### 2.1 仓库内客户交换 - 顺序交叉(OX)

**工作原理**:
1. 从parent1随机选择一段客户序列
2. 保持这段序列的相对顺序
3. 剩余位置按parent2的顺序填充
4. 客户在序列中的位置发生变化,导致路径分割时客户被分配到不同的路径

**示例**:
```
Parent1: [5, 3, 7, 2, 8, 1, 4, 6]
Parent2: [1, 2, 3, 4, 5, 6, 7, 8]
选择区间: [3, 5] (索引2-4)
Child:   [6, 1, 7, 2, 8, 3, 4, 5]
```

#### 2.2 跨仓库客户交换 - 边界客户重分配

**交换条件**:
- 客户到其他仓库的距离 < 当前仓库距离 × 0.8
- 目标仓库有足够容量
- 最多迭代5次

### 3. PSO的"速度"概念

**传统PSO速度更新公式**:
```
v(t+1) = w·v(t) + c1·r1·(pbest - x(t)) + c2·r2·(gbest - x(t))
x(t+1) = x(t) + v(t+1)
```

**离散PSO的速度模拟**:

通过概率选择体现速度:
- 惯性概率(prob_w): 保持当前搜索方向
- 认知概率(prob_c1): 向个体历史最优学习
- 社会概率(prob_c2): 向全局最优学习

**自适应惯性权重**:
```python
w = w_max - (w_max - w_min) * iteration / max_iterations
# 初期: w=0.9 (探索)
# 后期: w=0.4 (开发)
```

### 4. 混合初始化策略

**策略组成**:
- 30% 贪心构造 - 质量优先(最近邻启发式)
- 30% 随机构造 - 多样性优先
- 20% 最远插入 - 分散性优先
- 20% 节约算法 - 成本优先(Clarke-Wright)

**优势分析**:

| 策略 | 初始解质量 | 多样性 | 占比 |
|------|-----------|--------|------|
| 贪心构造 | ⭐⭐⭐⭐⭐ | ⭐⭐ | 30% |
| 随机构造 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 30% |
| 最远插入 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 20% |
| 节约算法 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 20% |

**实验数据对比** (P01):

| 初始化策略 | 第0代最优成本 | 第0代平均成本 | 最终成本 | 收敛代数 |
|-----------|--------------|--------------|---------|---------|
| 纯随机 | 850.2 | 950.5 | 625.3 | 300+ |
| 纯贪心 | 680.5 | 685.2 | 630.8 | 150 (局部最优) |
| 混合策略 | 746.6 | 930.5 | 616.2 | 200 |

### 5. 计算速度快的原因

**仓库分解策略**:
- 原问题: 4个仓库, 50个客户 → O(50!)
- 分解后: 4 × 12个客户 → O(4 × 12!)
- 复杂度大幅降低

**早停机制**:
- 配置: 200代没有改进则停止
- 实际效果: P01平均在150-200代停止,节省30-40%时间

**简化的速度更新**:
- 传统PSO: O(n²)
- 本实现: O(n)

## 第三部分: ACO算法机制详解

### 1. 初始化概述

ACO与GA有本质区别:
- GA需要初始化种群(多个完整解)
- ACO只需初始化信息素矩阵和启发式信息矩阵
- 蚂蚁在每次迭代中根据矩阵动态构造解

### 2. 信息素矩阵初始化

**初始化策略**: 基于最近邻启发式估算

```python
# 计算初始信息素浓度
tau_0 = 1.0 / (total_nodes * nn_cost)
```

**初始信息素值计算公式**:
τ₀ = 1 / (n × C_nn)

其中:
- n = 总节点数 (仓库数 + 客户数)
- C_nn = 最近邻启发式解的成本

**设计思想**:
- 使用最近邻启发式快速构造初始解
- 初始信息素值与问题规模成反比
- 所有边的初始信息素相同,确保探索公平性

### 3. 最近邻启发式

**算法流程**:
1. 从每个仓库出发
2. 贪心选择最近的未访问客户(满足容量约束)
3. 车辆满载后返回仓库
4. 累计总成本作为初始解估算

**P08示例**:
- 节点数: 251 (2仓库 + 249客户)
- 最近邻成本估算: ~5000-6000
- 初始信息素: τ₀ ≈ 0.0000007

### 4. 启发式信息矩阵初始化

**计算公式**:
η(i,j) = 1 / d(i,j)

其中:
- d(i,j) = 节点i到节点j的欧几里得距离
- 添加小常数1e-10避免除零错误

**设计思想**:
- 距离越近,启发式信息越大
- 体现"就近原则"的贪心思想
- 与信息素配合,平衡探索和开发

### 5. ACO vs GA 初始化对比

| 维度 | ACO | GA |
|------|-----|-----|
| 初始化对象 | 信息素矩阵 + 启发式矩阵 | 种群(多个完整解) |
| 初始化数量 | 2个矩阵(固定) | 100个个体(可变) |
| 初始化方式 | 数学公式计算 | 随机/启发式构造解 |
| 初始化成本 | O(n²) 矩阵计算 | O(种群规模 × 解构造成本) |
| 是否需要完整解 | 否 | 是 |
| 初始多样性 | 通过概率选择保证 | 通过随机初始化保证 |
| 初始质量 | 依赖最近邻估算 | 依赖构造启发式 |

### 6. ACO初始化的优缺点

**优点**:
1. 初始化简单快速,时间复杂度O(n²)
2. 理论基础扎实,基于最近邻启发式
3. 自然保证多样性,初始信息素均匀分布

**缺点**:
1. 无法利用高质量初始解
2. 冷启动问题,第0代完全依赖启发式信息
3. 初始信息素设置敏感

### 7. ACO初始化改进方向

**引入高质量初始解的信息素强化**:
```python
# 使用种子解强化初始信息素
for solution in seed_solutions:
    for route in solution['routes']:
        strengthen_pheromone_on_route(route, boost_factor=2.0)
```

**自适应初始信息素**:
- 小规模问题: 增加初始信息素
- 大规模问题: 降低初始信息素

**多样化启发式信息**:
- 综合考虑距离和需求
- η(i,j) = 1 / (distance + 1e-10) × (1.0 / (demand + 1.0))

## 算法性能总结

### 综合对比

| 特性 | GA | PSO | ACO |
|------|-----|-----|-----|
| 初始化策略 | RL混合(20%种子) | 混合(4种策略) | 信息素矩阵 |
| 求解质量 | 最优 | 良好 | 一般 |
| 计算速度 | 中等 | 快 | 慢 |
| 适用场景 | 高质量求解 | 实时应用 | 需改进 |
| 收敛速度 | 中等 | 快(早停) | 中等 |

### 实验结果验证

**P01数据对比**:
- GA: 平均Gap 5-7%, 时间 ~15秒
- PSO: 平均Gap 6.82%, 时间 2.87秒
- ACO: 平均Gap >30%, 时间 >30秒

**结论**:
- GA综合性能最优,RL混合初始化在大规模实例上优势明显
- PSO计算速度突出,混合初始化保证解质量稳定性
- ACO当前实现不推荐直接应用,需深度改进
