## 算法1：最佳位置插入算法

### 算法描述

在所有现有路径中寻找使增量成本最小的插入位置，若无法插入则尝试创建新路径。

### 伪代码

```
算法1: 最佳位置插入算法
输入: 待插入客户c, 目标仓库d, 路径集合R_d, 距离矩阵Dist, 容量约束Q, 里程约束L
输出: 更新后路径集合R_d', 插入标志flag, 增量成本Δ_best

1:  Δ_best ← +∞, best_route ← null, best_pos ← null
2:  for each r ∈ R_d do
3:      if ∑(demand[i] for i in r) + demand[c] > Q then continue  // 容量约束检查
4:      dist_r ← calculate_distance(r, d, Dist)
5:      for pos ← 1 to |r| + 1 do
6:          r' ← insert(r, c, pos)
7:          dist_r' ← calculate_distance(r', d, Dist)
8:          if dist_r' > L then continue  // 里程约束检查
9:          Δ ← dist_r' - dist_r
10:         if Δ < Δ_best then
11:             Δ_best ← Δ, best_route ← r, best_pos ← pos
12:         end if
13:     end for
14: end for
15: if best_route ≠ null then
16:     insert(best_route, c, best_pos), flag ← true  // 在最佳位置插入
17: else if demand[c] ≤ Q and 2 × Dist[d][c] ≤ L then
18:     R_d ← R_d ∪ {[c]}, Δ_best ← 2 × Dist[d][c], flag ← true  // 创建新路径
19: else
20:     flag ← false  // 插入失败
21: end if
22: return R_d, flag, Δ_best
```

### 复杂度分析

- **时间复杂度**: O(|R_d| × n²)
- **空间复杂度**: O(n)

### 关键说明

- **第3行**: 容量约束检查，确保插入后不超载
- **第8行**: 里程约束检查，确保插入后不超过最大行驶距离
- **第18行**: `{[c]}` 表示创建一条只包含客户c的新路径
  - 外层 `{}` 表示集合
  - 内层 `[]` 表示路径（列表）
  - 例如：R_d = {[1,2,3], [4,5]} ∪ {[6]} = {[1,2,3], [4,5], [6]}

## 算法2：边界客户跨仓库重分配

### 算法描述

识别位于仓库服务区域边界的客户，尝试将其重分配到距离更近的仓库，从而降低总配送成本。

### 伪代码

```
算法2: 边界客户跨仓库重分配
输入: 所有仓库集合D, 各仓库路径集合{R_d | d ∈ D}, 距离矩阵Dist, 
      容量约束Q, 里程约束L, 边界阈值θ
输出: 优化后的路径集合{R_d' | d ∈ D}, 总成本改善Δ_total

1:  Δ_total ← 0, improved ← true
2:  while improved do
3:      improved ← false
4:      for each 仓库对 (d_i, d_j) ∈ D × D, i ≠ j do
5:          BC ← identify_boundary_customers(d_i, d_j, R_{d_i}, Dist, θ)  // 识别边界客户
6:          for each 边界客户 c ∈ BC do
7:              cost_remove ← calculate_removal_cost(c, R_{d_i}, d_i, Dist)  // 移除成本
8:              R_{d_j}', flag, Δ_insert ← 算法1(c, d_j, R_{d_j}, Dist, Q, L)  // 使用最佳位置插入
9:              if flag = true and Δ_insert < cost_remove then  // 插入成功且有益
10:                 remove_customer(c, R_{d_i})  // 从原仓库移除
11:                 R_{d_j} ← R_{d_j}'  // 更新目标仓库路径
12:                 Δ_total ← Δ_total + (cost_remove - Δ_insert)
13:                 improved ← true
14:             end if
15:         end for
16:     end for
17: end while
18: return {R_d' | d ∈ D}, Δ_total
```

### 边界客户识别函数

```
函数: identify_boundary_customers
输入: 源仓库d_i, 目标仓库d_j, 源仓库路径集合R_{d_i}, 距离矩阵Dist, 边界阈值θ
输出: 边界客户集合BC

1:  BC ← ∅
2:  for each 路径 r ∈ R_{d_i} do
3:      for each 客户 c ∈ r do
4:          if Dist[d_j][c] < Dist[d_i][c] × (1 + θ) then  // 到目标仓库距离接近
5:              BC ← BC ∪ {c}
6:          end if
7:      end for
8:  end for
9:  return BC
```

### 复杂度分析

- **时间复杂度**: O(|D|² × |BC| × |R_d| × n²)
- **空间复杂度**: O(|BC|)

### 参数说明

- **θ (theta)**: 边界阈值，通常取0.1-0.2
  - θ = 0.1 表示第二近仓库距离在最近仓库的1.1倍以内
  - θ 越大，边界范围越宽，候选客户越多

### 关键说明

- **第5行**: 识别边界客户 - 到目标仓库距离接近当前仓库的客户
- **第8行**: 调用算法1执行最佳位置插入，自动处理容量和里程约束
- **第9行**: 只有插入成功且成本降低时才执行重分配

## 算法3：2-opt局部搜索

### 算法描述

通过反转路径中的一段来消除路径交叉，降低路径总长度。这是一种经典的路径内优化方法。

### 伪代码

```
算法3: 2-opt局部搜索（路径内优化）
输入: 路径r, 仓库d, 距离矩阵Dist, 最大迭代次数max_iter
输出: 优化后的路径r'

1:  improved ← true, iter ← 0
2:  while improved and iter < max_iter do
3:      improved ← false
4:      for i ← 1 to |r| - 2 do
5:          for j ← i + 2 to |r| do
6:              r' ← reverse(r, i, j)  // 反转r[i+1:j]
7:              if cost(r') < cost(r) then
8:                  r ← r'
9:                  improved ← true
10:                 break
11:             end if
12:         end for
13:         if improved then break
14:     end for
15:     iter ← iter + 1
16: end while
17: return r
```

### 复杂度分析

- **时间复杂度**: O(n² × iter)
- **空间复杂度**: O(n)

### 优化版本（ACO中使用）

```
算法3-优化版: 2-opt局部搜索（带早停和窗口限制）
输入: 路径r
输出: 优化后的路径r'

1:  max_iterations ← 3  // 限制迭代次数
2:  max_window ← 20  // 限制搜索窗口
3:  for iteration ← 1 to max_iterations do
4:      improved ← false
5:      for i ← 1 to |r| - 1 do
6:          for j ← i + 2 to min(i + max_window, |r|) do
7:              r' ← reverse(r, i, j)  // 2-opt交换
8:              if cost(r') < cost(r) then
9:                  r ← r'
10:                 improved ← true
11:                 break  // 早停：找到改进立即应用
12:             end if
13:         end for
14:         if improved then break
15:     end for
16:     if not improved then break  // 无改进则停止
17: end for
18: return r
```

### 关键说明

- **第6行**: 反转操作 - 将路径片段 r[i+1:j] 反转
- **第10行**: 早停机制 - 找到改进立即应用，提高效率
- **优化版第2行**: 窗口限制 - 只在局部范围内搜索，减少计算量

## 算法4：路径间客户交换

### 算法描述

在同一仓库的不同路径之间交换客户，或在不同仓库的路径之间转移客户，以降低总成本。

### 伪代码（同仓库路径间交换）

```
算法4: 路径间客户交换（同仓库路径间优化）
输入: 路径集合R_d, 仓库d, 距离矩阵Dist, 容量约束Q, 里程约束L
输出: 优化后的路径集合R_d'

1:  improved ← true
2:  while improved do
3:      improved ← false
4:      for each 路径对 (r_i, r_j) ∈ R_d × R_d, i ≠ j do
5:          for each 客户 c_i ∈ r_i do
6:              for each 客户 c_j ∈ r_j do
7:                  cost_before ← cost(r_i) + cost(r_j)
8:                  r_i', r_j' ← swap(r_i, c_i, r_j, c_j)  // 交换客户
9:                  if feasible(r_i', Q, L) and feasible(r_j', Q, L) then
10:                     cost_after ← cost(r_i') + cost(r_j')
11:                     if cost_after < cost_before then
12:                         r_i ← r_i', r_j ← r_j'
13:                         improved ← true
14:                         break
15:                     end if
16:                 end if
17:             end for
18:             if improved then break
19:         end for
20:         if improved then break
21:     end for
22: end while
23: return R_d
```

### 复杂度分析

- **时间复杂度**: O(|R|² × n²)
- **空间复杂度**: O(n)

### 关键说明

- **第8行**: swap操作 - 交换两条路径中的客户
- **第9行**: 可行性检查 - 确保交换后两条路径都满足容量和里程约束
- **第11行**: 成本改进检查 - 只有总成本降低时才接受交换

## 算法组合策略

### 典型的MDVRP求解流程

```
阶段1: 初始解构造
  ├─ 客户-仓库分配（最近仓库分配）
  └─ 路径构造（算法1：最佳位置插入）

阶段2: 仓库分配优化 ⭐
  └─ 算法2：边界客户跨仓库重分配

阶段3: 路径优化
  ├─ 算法3：2-opt局部搜索（路径内优化）
  └─ 算法4：路径间客户交换（路径间优化）

阶段4: 全局优化（遗传算法）
  ├─ 选择、交叉、变异
  ├─ 变异中使用：边界客户跨仓库重分配
  └─ 局部搜索：2-opt + 路径间交换
```

### 算法调用关系

```
遗传算法主循环
  │
  ├─ 初始种群生成
  │   ├─ 最近仓库分配
  │   ├─ 算法1（最佳位置插入）
  │   └─ 算法2（边界客户跨仓库重分配）
  │
  ├─ 选择操作
  │
  ├─ 交叉操作
  │   └─ 算法1（修复不可行解）
  │
  ├─ 变异操作
  │   ├─ 路径内变异（2-opt）
  │   └─ 跨仓库变异（边界客户重分配）
  │
  └─ 局部搜索（每N代执行）
      ├─ 算法3（2-opt优化）
      ├─ 算法4（路径间交换）
      └─ 算法2（边界客户重分配）
```

class MDVRPSolver:
    def __init__(self):
        self.insertion = BestInsertionAlgorithm()
        self.boundary = BoundaryCustomerReassignment()
        self.two_opt = TwoOptOptimizer()
        self.exchange = InterRouteExchange()
    
    def solve(self, problem):
        # 阶段1: 初始解构造

        # 阶段2: 仓库分配优化
        solution = self.boundary.optimize(solution)
        
        # 阶段3: 路径优化

        return solution

## 总结

本文档整理的4个算法构成了完整的MDVRP优化策略体系：

1. **算法1（最佳位置插入）**: 基础构造算法，被其他算法调用
2. **算法2（边界客户跨仓库重分配）**: 仓库分配层面的优化
3. **算法3（2-opt局部搜索）**: 路径内部的优化
4. **算法4（路径间客户交换）**: 路径之间的协同优化

这些算法相互配合，从不同层次优化MDVRP解的质量，形成了一个层次化、模块化的优化框架。

### 核心发现

1. **最佳插入算法是保证解质量的基础**
2. **边界客户重分配对多仓库问题至关重要**（改善2-5%）
3. **2-opt局部搜索在小规模问题上非常有效**（改善5-15%）
4. **路径间交换在负载不均时效果明显**（改善3-8%）

### 最佳实践

- **小规模**: 最佳插入 + 2-opt
- **中规模**: 边界客户重分配 + 2-opt + 路径间交换
- **大规模**: 边界客户重分配 + 2-opt（限制迭代）
- **实时系统**: 快速模式（减少迭代次数）
- **离线优化**: 精细模式（增加迭代次数）
