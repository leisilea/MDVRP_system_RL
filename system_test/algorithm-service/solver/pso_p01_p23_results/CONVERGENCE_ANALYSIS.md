# PSO收敛曲线分析 - P01

## 数据处理标准

### 合并逻辑
1. **识别仓库边界**: 当generation值回退时，识别为新仓库的数据开始
2. **按generation对齐**: 相同generation的best_cost和avg_cost分别相加
3. **处理不同迭代次数**: 如果某仓库提前收敛，使用其最后一个generation的值填充后续generation

### 平均值计算逻辑（配置对比图）
对于每个配置（small/medium/large），计算所有run的平均收敛曲线时：
- 如果某个run在generation X有数据，直接使用
- 如果某个run在generation X没有数据（提前收敛），使用该run的最后一个generation的值
- 这样确保每个run对所有generation的平均值都有贡献，避免曲线异常波动

## P01结果总结

### Small配置
- 3个runs，每个run都迭代到generation 300
- 平均最终成本: 618.51
- 初始成本范围: 731.71 - 746.60
- 最终成本范围: 617.97 - 618.80

### Medium配置
- 3个runs，迭代到generation 250-300
- 平均最终成本: 618.88
- 初始成本范围: 691.27 - 748.98
- 最终成本范围: 615.76 - 623.81
- Run 1表现最好: 615.76

### Large配置
- 3个runs，迭代到generation 200-350
- 平均最终成本: 618.95
- 初始成本范围: 693.09 - 736.57
- 最终成本范围: 612.81 - 626.99
- Run 1表现最好: 612.81
- Run 2提前收敛（generation 200）

## 关键发现

### 1. 配置对比
- Large配置的Run 1达到了最优成本（612.81），最接近BKS（576.87）
- 但Large配置的平均成本（618.95）略高于Small（618.51）和Medium（618.88）
- 这是因为Large配置的Run 2表现较差（626.99）

### 2. 收敛特性
- 所有配置都在前100代快速收敛
- Generation 0-50: 成本快速下降
- Generation 50-200: 继续优化，下降速度减缓
- Generation 200+: 基本稳定，微小改进

### 3. 早停机制
- Run 2 (Large配置) 在generation 200提前停止
- 说明早停机制正常工作（200代无改进）
- 使用最后值填充策略确保了平均曲线的平滑性

### 4. 与BKS的差距
- 最优结果: 612.81 (Large Run 1)
- BKS: 576.87
- Gap: 6.23%
- 平均gap: 约7.0-7.3%

## 建议

1. **参数调优**: Large配置虽然迭代次数最多，但平均表现不如Small和Medium，可能需要调整粒子数或其他参数
2. **多次运行**: 增加运行次数可以更准确地评估算法性能
3. **早停阈值**: 当前200代无改进即停止，可以考虑调整这个阈值
4. **跨仓库优化**: 可以进一步优化跨仓库边界客户交换策略
