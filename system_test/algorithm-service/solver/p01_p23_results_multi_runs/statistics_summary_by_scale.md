### 按规模对比

|------|--------|-----------------|----------------|---------|------------------|-----------------|----------|

### 关键发现

1. **规模效应**：
   - Hybrid方法在大规模算例上改进最显著（平均降低0.80个百分点）
   - 小规模算例也有一定改进（平均降低0.28个百分点）
   - 中规模算例改进不明显（略有退化0.05个百分点）

2. **时间开销**：
   - 小规模算例时间开销最大（+28.7%），主要是RL模型初始化开销
   - 大规模算例时间开销最小（+5.9%），RL初始化开销被摊薄
   - 总体时间增加约9.4%，在可接受范围内

3. **最佳改进案例**：
   - P04: 8.95% → 5.44% (改进3.51个百分点)
   - P08: 7.89% → 5.70% (改进2.19个百分点)
   - P11: 12.17% → 10.04% (改进2.13个百分点)
   - P10: 8.54% → 6.54% (改进2.00个百分点)

4. **退化案例**：
   - P21: 13.25% → 14.84% (退化1.59个百分点) - 最大规模算例，可能需要更多迭代
   - P16: 2.30% → 2.55% (退化0.25个百分点)

5. **稳定性**：
   - 23个算例中，15个算例有改进，6个算例性能相同或略有退化，2个算例明显退化
   - 改进率：65.2%

### 结论

强化学习辅助初始化（Hybrid方法）在MDVRP问题上整体有效，特别是在大规模算例上表现出色。虽然增加了约9.4%的计算时间，但平均Gap降低了0.46个百分点，在大规模算例上降低了0.80个百分点。该方法在249客户规模的算例（P08-P11）上改进最为显著，证明了强化学习在复杂问题上的优势。
