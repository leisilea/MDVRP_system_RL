# 简化版重规划方案

## 背景

原有的重规划方案使用临时仓库（将车辆当前位置转换为临时仓库）进行全局重规划，但效果不理想：
- 容量约束难以满足（车辆剩余容量不足）
- 需要复杂的PSO求解器
- 计算时间长
- 结果质量不稳定

## 新方案：单车辆局部重规划

### 核心思想

当检测到路段堵塞时，只对受影响的单个车辆进行局部重规划，而不是全局重规划。

### 重规划流程

1. **识别受影响车辆**
   - 检查每条路径是否包含被堵塞的路段
   - 考虑车辆当前位置，只检查未服务部分的路径
   - 标记所有受影响的车辆

2. **局部重规划**
   - 对每个受影响的车辆：
     - 确定未服务客户列表（当前位置之后的客户）
     - 使用贪心最近邻算法重新排序这些客户
     - 避开被堵塞的路段
   - 不受影响的车辆保持原路径不变

3. **返回结果**
   - 返回所有路径（修改后的 + 未修改的）
   - 标记哪些车辆被重规划了

### 算法：贪心最近邻（Greedy Nearest Neighbor）

```
输入：
- 起点位置（车辆当前位置或仓库）
- 未服务客户列表
- 阻塞路段集合

算法：
1. 从起点开始
2. 重复直到所有客户都被访问：
   a. 在剩余客户中找到最近的客户（且不经过阻塞路段）
   b. 如果所有客户都被阻塞，选择距离最近的（忽略阻塞）
   c. 将该客户添加到路径
   d. 更新当前位置为该客户
3. 返回新路径

输出：
- 重新排序后的客户访问顺序
```

### 优点

1. **简单高效**
   - 不需要复杂的求解器
   - 计算时间短（O(n²)，n为未服务客户数）
   - 易于理解和调试

2. **局部性**
   - 只影响受堵塞影响的车辆
   - 其他车辆保持原计划
   - 减少对整体方案的干扰

3. **实用性**
   - 不需要考虑容量约束（车辆已经在执行中，容量已确定）
   - 不需要临时仓库转换
   - 符合实际场景（司机只需要知道新的访问顺序）

### 实现文件

- `service_simple.py`: 简化版重规划服务
  - `SimpleReplanningService.replan()`: 主入口
  - `_identify_affected_routes()`: 识别受影响车辆
  - `_greedy_reorder()`: 贪心重排序算法

### 数据流

```
输入：
├── depots: 仓库列表
├── customers: 客户列表
├── routes: 当前路径列表
├── blocked_edges: 阻塞路段列表
└── vehicle_positions: 车辆当前位置 {vehicle_id: customer_id}

处理：
├── 识别受影响车辆
│   └── 检查路径是否包含阻塞路段
├── 对每个受影响车辆
│   ├── 提取未服务客户
│   └── 贪心重排序
└── 保持其他车辆不变

输出：
├── new_routes: 所有路径（修改后 + 未修改）
├── replanned_route_ids: 被重规划的车辆ID列表
└── solve_time: 求解时间
```

### 示例场景

**原始路径：**
- 车辆1: 仓库 → 客户A → 客户B → 客户C → 仓库
- 车辆2: 仓库 → 客户D → 客户E → 仓库

**堵塞情况：**
- 路段 A→B 被堵塞
- 车辆1当前位置：客户A

**重规划结果：**
- 车辆1: 客户A → 客户C → 客户B → 仓库（重新排序，避开A→B）
- 车辆2: 仓库 → 客户D → 客户E → 仓库（不受影响，保持不变）

### 与原方案对比

| 特性 | 原方案（临时仓库） | 新方案（局部重规划） |
|------|-------------------|---------------------|
| 复杂度 | 高（需要PSO求解器） | 低（贪心算法） |
| 计算时间 | 长（5-10秒） | 短（<1秒） |
| 容量约束 | 难以满足 | 不需要考虑 |
| 影响范围 | 全局（所有车辆） | 局部（受影响车辆） |
| 结果质量 | 不稳定 | 稳定 |
| 实用性 | 低 | 高 |

### 后续改进方向

1. **算法优化**
   - 使用2-opt或3-opt改进贪心解
   - 考虑多个堵塞路段的组合影响
   - 动态调整路径（实时重规划）

2. **多车辆协同**
   - 如果单车辆无法完成，考虑将客户转移给其他车辆
   - 车辆之间的任务交换

3. **智能决策**
   - 评估绕行 vs 等待的成本
   - 预测堵塞持续时间
   - 提供多个重规划方案供选择
