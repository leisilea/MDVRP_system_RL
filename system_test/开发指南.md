## 项目概述

### 系统定位
这是一个**物流调度决策支持系统**，而非传统的无人驾驶路径规划（CV问题）。重点在于：
- 运筹优化算法的实现与验证
- 多车辆、多仓库的任务分配
- 调度方案的可视化演示
- 算法性能评估与对比

### 核心功能
1. **场景配置**: 创建/编辑仓库、客户、车辆参数
2. **算法求解**: 调用Python算法服务计算最优路径
3. **结果可视化**: 地图展示路径、统计图表、动态演示
4. **方案对比**: 对比不同算法的性能指标

## 微服务集成方案

### 为什么选择微服务架构？

1. **技术解耦**: Java处理业务逻辑，Python专注算法计算
2. **独立部署**: 两个服务可以独立更新、扩展
3. **语言优势**: 发挥各语言生态优势
4. **行业标准**: 符合现代软件工程实践

### 三种集成方案对比

| 方案 | 实现方式 | 优点 | 缺点 | 推荐度 |
|------|---------|------|------|--------|

```java
@Service
public class AlgorithmService {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @Value("${algorithm.service.url}")
    private String algorithmServiceUrl;
    
    /**
     * 调用Python算法服务求解MDVRP
     */
    public SolutionDTO solve(Long scenarioId, AlgorithmParams params) {
        // 1. 从数据库读取场景数据
        List<Depot> depots = depotMapper.selectByScenarioId(scenarioId);
        List<Customer> customers = customerMapper.selectByScenarioId(scenarioId);
        
        // 2. 转换为DTO
        ScenarioDTO scenarioDTO = new ScenarioDTO();
        scenarioDTO.setDepots(convertDepots(depots));
        scenarioDTO.setCustomers(convertCustomers(customers));
        scenarioDTO.setParams(params);
        
        // 3. 调用Python服务
        String url = algorithmServiceUrl + "/api/solve";
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<ScenarioDTO> request = new HttpEntity<>(scenarioDTO, headers);
        
        ResponseEntity<SolutionDTO> response = restTemplate.postForEntity(
            url, request, SolutionDTO.class
        );
        
        // 4. 保存结果到数据库
        if (response.getBody().getSuccess()) {
            saveSolution(scenarioId, response.getBody());
        }
        
        return response.getBody();
    }
}
```

### 3. Python算法封装标准模板

```python
# solver/mdvrp_solver.py

class MDVRPSolver:
    """MDVRP求解器基类"""
    
    def __init__(self, depots, customers, vehicles):
        self.depots = depots
        self.customers = customers
        self.vehicles = vehicles
        self.distance_matrix = self._calculate_distance_matrix()
    
    def _calculate_distance_matrix(self):
        """计算距离矩阵"""
        n = len(self.customers)
        matrix = np.zeros((n, n))
        
        for i in range(n):
            for j in range(n):
                if i != j:
                    matrix[i][j] = self._euclidean_distance(
                        self.customers[i], self.customers[j]
                    )
        
        return matrix
    
    def _euclidean_distance(self, p1, p2):
        """欧几里得距离"""
        return np.sqrt((p1['x'] - p2['x'])**2 + (p1['y'] - p2['y'])**2)
    
    def solve(self):
        """求解方法（由子类实现）"""
        raise NotImplementedError


# solver/genetic_algorithm.py

class GeneticAlgorithm(MDVRPSolver):
    """遗传算法实现"""
    
    def __init__(self, depots, customers, vehicles, params):
        super().__init__(depots, customers, vehicles)
        self.population_size = params.get('population_size', 50)
        self.max_generations = params.get('max_iterations', 1000)
        self.mutation_rate = params.get('mutation_rate', 0.1)
    
    def solve(self):
        # 初始化种群
        population = self._initialize_population()
        
        best_solution = None
        best_fitness = float('inf')
        
        for generation in range(self.max_generations):
            # 计算适应度
            fitness = [self._evaluate(ind) for ind in population]
            
            # 更新最优解
            min_idx = np.argmin(fitness)
            if fitness[min_idx] < best_fitness:
                best_fitness = fitness[min_idx]
                best_solution = population[min_idx]
            
            # 选择、交叉、变异
            population = self._evolve(population, fitness)
        
        # 转换为标准格式
        return self._format_solution(best_solution, best_fitness)
    
    def _format_solution(self, solution, cost):
        """将内部解格式转换为API返回格式"""
        routes = []
        
        for vehicle_id, path in enumerate(solution):
            routes.append({
                'vehicle_id': vehicle_id + 1,
                'depot_id': 1,  # 根据实际情况分配
                'path': path,
                'cost': self._calculate_route_cost(path)
            })
        
        return {
            'routes': routes,
            'total_cost': cost,
            'compute_time': 0  # 由外层记录
        }
```

### 常见答辩问题

#### Q1: 为什么选择微服务架构而不是单体应用？
**A**: 
1. 技术解耦：Java处理业务逻辑，Python专注算法，发挥各自优势
2. 独立扩展：算法服务可以独立部署到高性能服务器
3. 符合工业标准：真实企业也采用类似架构

#### Q2: 如何保证Java和Python服务通信的可靠性？
**A**:
1. 健康检查：定期检测Python服务是否可用
2. 超时重试：设置合理超时时间，失败后自动重试
3. 异常处理：捕获网络异常并返回友好提示
4. 日志记录：记录每次调用的详细日志

#### Q3: 算法的时间复杂度是多少？
**A**: 
- 遗传算法：O(G × P × N²)
  - G: 迭代次数
  - P: 种群大小
  - N: 客户数量
- 实际测试：100客户约10-15秒

#### Q4: 如何验证算法的正确性？
**A**:
1. 使用标准测试集（Cordeau MDVRP Benchmark）
2. 与已发表论文结果对比
3. 检查约束满足（载重、时间窗）
4. 可视化检查路径合理性

#### Q5: 前端地图为什么选择百度地图而不是 Leaflet/高德地图？
**A**:
1. **真实路网数据**：百度地图提供真实的中国道路、建筑、POI 数据，演示更贴近实际物流场景
2. **国内优化**：在中国境内访问速度快，瓦片加载稳定
3. **丰富功能**：内置地理编码、路径规划、热力图等高级功能
4. **易于集成**：API 简单，中文文档完善，适合毕设快速开发
