# 元启发式算法优化策略对比

## 概述

本文档对比分析PSO、ACO、GA、GA+RL四种元启发式算法在MDVRP问题中使用的优化策略（局部优化算子），并基于P01-P23测试结果分析各策略的有效性。

---

## 1. 遗传算法 (GA)

### 1.1 核心优化策略

| 策略类型 | 算子名称 | 实现方式 | 作用 |
|---------|---------|---------|------|
| **选择算子** | 锦标赛选择 | Tournament Selection | 选择优秀个体进入下一代 |
| **交叉算子** | 最佳成本路径交叉 | Best Cost Route Crossover | 组合父代优势生成子代 |
| **变异算子** | 反转变异 | Reversal Mutation | 反转路径片段，探索新解 |
| **变异算子** | 重定位变异 | Relocation Mutation | 客户重新插入最佳位置 |
| **变异算子** | 跨仓库变异 | Inter-Depot Swapping | 边界客户跨仓库重分配 |
| **精英策略** | 精英保留 | Elitism | 保留最优个体 |
| **模式切换** | 精细化模式 | Refinement Mode | 后期切换为重定位变异 |

### 1.2 详细说明

#### 1.2.1 最佳成本路径交叉 (BCRC)
```
输入: 父代p1, p2, 交叉率crossoverRate
输出: 子代offspring1, offspring2

1. if random() > crossoverRate: return p1, p2  // 按概率决定是否交叉
2. 随机选择一个仓库d
3. 从p1和p2中各随机选择一条路径r1, r2
4. 从p2中移除r1中的所有客户
5. 从p1中移除r2中的所有客户
6. 将r1中的客户逐个插入p2的最佳位置 → offspring2
7. 将r2中的客户逐个插入p1的最佳位置 → offspring1
8. return offspring1, offspring2
```

#### 1.2.2 反转变异 (Reversal)
```
输入: 个体individual
输出: 变异后的individual

1. 随机选择一个仓库d
2. 将该仓库的所有路径展平为客户序列S
3. 随机选择两个切点i, j (i < j)
4. 反转S[i:j]片段
5. 将反转后的序列重新分配回原路径结构
```

#### 1.2.3 重定位变异 (Relocation)
```
输入: 个体individual
输出: 优化后的individual

1. 随机选择一个仓库d
2. 从该仓库随机选择一个客户c
3. 从当前路径中移除c
4. 将c插入到该仓库的最佳位置（算法2-5）
```

#### 1.2.4 跨仓库变异 (Inter-Depot Swapping)
```
输入: 个体individual
输出: 优化后的individual

1. 识别所有边界客户（可服务于多个仓库的客户）
2. 随机选择一个边界客户c
3. 从当前仓库d1移除c
4. 从c的可服务仓库列表中随机选择另一个仓库d2
5. 将c插入到d2的最佳位置（算法2-5）
```

### 1.3 参数配置

| 参数 | 值 | 说明 |
|-----|-----|-----|
| 种群大小 | 100 | 个体数量 |
| 交叉率 | 0.8 | 交叉概率 |
| 变异率 | 0.2 | 变异概率 |
| 精英保留数 | 2 | 保留最优个体数 |
| 跨仓库频率 | 每10代 | 执行跨仓库变异的频率 |
| 精细化阈值 | 100代后 | 切换到重定位变异的代数 |

## 2. 遗传算法+强化学习 (GA+RL)

### 2.1 核心优化策略

GA+RL = **GA的所有策略** + **RL初始化**

| 策略类型 | 算子名称 | 实现方式 | 作用 |
|---------|---------|---------|------|
| **初始化** | RL采样初始化 | RL4CO模型采样 | 生成高质量初始种群 |
| 其他策略 | 同GA | 同GA | 同GA |

### 2.2 RL初始化详细说明

```
输入: MDVRP实例, 种群大小N
输出: 初始种群population

1. 使用RL4CO预训练模型
2. 采样N个解（使用贪婪解码或采样解码）
3. 将RL生成的解转换为GA染色体格式
4. return population
```

### 2.3 与Pure GA的区别

| 维度 | Pure GA | GA+RL |
|-----|---------|-------|
| 初始化方式 | 随机初始化 | RL模型采样 |
| 初始解质量 | 较低 | 较高 |
| 初始化时间 | 快（< 1秒） | 慢（10-40秒） |
| 收敛速度 | 慢 | 快 |
| 最终解质量 | 中等 | 较好 |

## 3. 粒子群优化 (PSO)

### 3.1 核心优化策略

| 策略类型 | 算子名称 | 实现方式 | 作用 |
|---------|---------|---------|------|
| **初始化** | 多样化初始化 | 贪心+最远插入+节约算法 | 生成多样化初始种群 |
| **更新算子** | 速度-位置更新 | PSO标准公式 | 粒子向最优解移动 |
| **交叉算子** | 路径级交叉 | Route-Level Crossover | 交换完整路径 |
| **局部搜索** | 重定位优化 | Relocation | 客户重新插入最佳位置 |
| **局部搜索** | 边界客户优化 | Boundary Customer Opt | 边界客户跨仓库重分配 |
| **全局优化** | 跨仓库客户交换 | Inter-Depot Swap | 全局边界客户交换 |

### 3.2 详细说明

#### 3.2.1 多样化初始化
```
输入: 客户列表, 种群大小N
输出: 初始种群population

for i in range(N):
    if i % 3 == 0:
        particle = greedy_construct_route()  // 贪心构造
    elif i % 3 == 1:
        particle = farthest_insertion()  // 最远插入
    else:
        particle = savings_algorithm()  // 节约算法
    population.append(particle)
return population
```

#### 3.2.2 重定位优化
```
输入: 粒子particle, 重定位次数K
输出: 优化后的particle

for k in range(K):
    1. 解码particle为路径集合routes
    2. 随机选择一个客户c
    3. 从routes中移除c
    4. 找到c的最佳插入位置
    5. 插入c到最佳位置
    6. 编码routes回particle
return particle
```

#### 3.2.3 边界客户优化
```
输入: 粒子particle, 仓库坐标
输出: 优化后的particle

1. 解码particle为路径集合routes
2. 识别边界客户: dist_to_other < dist_to_current * 0.8
3. 随机选择一个边界客户c
4. 从routes中移除c
5. 重新插入c到最佳位置
6. 编码routes回particle
return particle
```

### 3.3 参数配置

| 参数 | 值 | 说明 |
|-----|-----|-----|
| 粒子数量 | 30 | 种群大小 |
| 惯性权重 | 0.7 | 保持原速度的比例 |
| 认知系数 | 1.5 | 个体最优影响 |
| 社会系数 | 1.5 | 全局最优影响 |
| 重定位次数 | 3 | 每次优化的重定位次数 |

## 4. 蚁群优化 (ACO)

### 4.1 核心优化策略

| 策略类型 | 算子名称 | 实现方式 | 作用 |
|---------|---------|---------|------|
| **构造算子** | 概率选择 | 信息素+启发式 | 构造解 |
| **启发式** | 最近邻估计 | Nearest Neighbor | 初始化信息素 |
| **局部搜索** | 2-opt优化 | 2-opt with Early Stop | 路径优化 |
| **信息素更新** | 全局更新 | 精英蚂蚁策略 | 强化优秀路径 |
| **信息素蒸发** | 蒸发机制 | ρ衰减 | 避免过早收敛 |

### 4.2 详细说明

#### 4.2.1 概率选择构造解
```
输入: 当前客户i, 候选客户集合N
输出: 下一个客户j

for j in N:
    τ_ij = pheromone[i][j]  // 信息素
    η_ij = 1 / distance[i][j]  // 启发式信息
    p_ij = (τ_ij^α) * (η_ij^β)  // 选择概率
    
j = roulette_wheel_selection(p)  // 轮盘赌选择
return j
```

#### 4.2.2 2-opt局部搜索（优化版）
```
输入: 路径route
输出: 优化后的route

max_iterations = 3  // 限制迭代次数
max_window = 20  // 限制搜索窗口

for iteration in range(max_iterations):
    improved = False
    for i in range(len(route) - 1):
        for j in range(i + 2, min(i + max_window, len(route))):
            new_route = route[:i] + route[i:j+1][::-1] + route[j+1:]  // 2-opt交换
            if cost(new_route) < cost(route):
                route = new_route
                improved = True
                break  // 早停：找到改进立即应用
        if improved: break
    if not improved: break  // 无改进则停止
return route
```

#### 4.2.3 信息素更新
```
输入: 所有解solutions, 成本costs
输出: 更新后的pheromone

1. 信息素蒸发: pheromone *= (1 - ρ)
2. 精英蚂蚁更新:
   for solution in top_k_solutions:
       for edge (i,j) in solution:
           pheromone[i][j] += Q / cost(solution)
```

### 4.3 参数配置

| 参数 | 值 | 说明 |
|-----|-----|-----|
| 蚂蚁数量 | 20 | 每代构造的解数量 |
| α (信息素权重) | 1.0 | 信息素影响因子 |
| β (启发式权重) | 2.0 | 距离启发式因子 |
| ρ (蒸发率) | 0.1 | 信息素蒸发系数 |
| Q (信息素强度) | 100 | 信息素增量系数 |
| 2-opt迭代次数 | 3 | 局部搜索迭代上限 |
| 2-opt窗口大小 | 20 | 搜索窗口限制 |

| 优化目标 | 推荐算法 | 原因 |

## 8. 改进方向

### 8.1 GA/GA+RL改进
1. **自适应参数**: 根据收敛情况动态调整交叉率和变异率
2. **增强局部搜索**: 添加2-opt/3-opt（借鉴ACO）
3. **多样性维护**: 引入多样性度量，防止早熟收敛

### 8.2 PSO改进
1. **惯性权重自适应**: 根据迭代动态调整
2. **混合局部搜索**: 结合2-opt和重定位
3. **多种群策略**: 维护多个子种群

### 8.3 ACO改进
1. **加速收敛**: 减少蚂蚁数量，增加精英蚂蚁权重
2. **混合初始化**: 结合RL初始化
3. **并行化**: 多进程构造解

---

## 9. 结论

### 9.1 核心发现

1. **RL初始化是最有效的策略**（中大规模问题gap改善1-2%）
2. **跨仓库变异对多仓库问题至关重要**
3. **2-opt局部搜索在小规模问题上非常有效**
4. **精细化模式切换（反转→重定位）提升后期收敛**
5. **最佳插入算法是保证解质量的基础**

### 9.2 最佳实践

- **小规模**: Pure GA + 快速收敛策略
- **中规模**: GA+RL + 跨仓库变异
- **大规模**: GA+RL + 精细化模式
- **实时系统**: Pure GA（速度优先）
- **离线优化**: GA+RL（质量优先）

### 9.3 未来方向

1. 开发自适应混合策略（根据问题规模自动选择）
2. 优化RL初始化效率（目标：减少50%时间开销）
3. 探索更多局部搜索算子（3-opt, Or-opt）
4. 研究多目标优化策略（时间-质量-车辆数）
