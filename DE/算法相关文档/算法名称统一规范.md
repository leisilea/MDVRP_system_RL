# 算法名称统一规范

## 概述

本文档定义了系统中所有算法的统一命名规范,确保前端、后端、Flask服务和数据库中使用一致的算法名称。

## 标准算法名称

系统支持以下5种算法,名称必须严格按照以下规范:

| 算法名称 | 显示名称 | 说明 | 颜色标签 |
|---------|---------|------|---------|
| `genetic` | 遗传算法 (GA-Java) | 基于Java的遗传算法实现 (Ombuki-Berman 2009) | 蓝色 (primary) |
| `ga_multiprogramming` | 遗传算法多进程 (GA-Python) | 基于Python的多进程遗传算法实现 | 橙色 (warning) |
| `GA_RL_HYBRID` | 遗传算法混合强化学习 (GA-RL) | 结合遗传算法和强化学习(RouteFinder)的混合求解器 | 绿色 (success) |
| `ACO` | 蚁群算法 (ACO) | 蚁群优化算法 (Ant Colony Optimization) | 浅蓝色 (info) |
| `PSO` | 粒子群算法 (PSO) | 粒子群优化算法 (Particle Swarm Optimization) | 红色 (danger) |

### 3. Flask 算法服务

#### app.py
```python
# 验证算法参数
algorithm = params.get('algorithm', 'genetic')
valid_algorithms = ['genetic', 'ga_multiprogramming', 'GA_RL_HYBRID', 'ACO', 'PSO']
if algorithm not in valid_algorithms:
    raise ValueError(f"不支持的算法: {algorithm}，支持的算法: {', '.join(valid_algorithms)}")
```

#### mdvrp_solver.py
```python
def create_solver(depots, customers, params):
    algorithm = params.get('algorithm', 'genetic')
    
    if algorithm == 'genetic':
        return GeneticAlgorithmSolver(depots, customers, params)
    elif algorithm == 'ga_multiprogramming':
        return GAMultiprogrammingSolver(depots, customers, params)
    elif algorithm == 'ACO':
        from . import aco
        return aco.AntColonySolver(depots, customers, params)
    elif algorithm == 'PSO':
        from . import pso
        return pso.ParticleSwarmSolver(depots, customers, params)
    elif algorithm == 'GA_RL_HYBRID':
        return GAMDVRPRLHybridSolver(depots, customers, params)
    else:
        raise ValueError(f"不支持的算法类型: {algorithm}")
```

### 4. 数据库 (MySQL)

#### solution 表
```sql
CREATE TABLE solution (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    scenario_id BIGINT NOT NULL,
    algorithm VARCHAR(50) NOT NULL,  -- 存储: genetic, ga_multiprogramming, GA_RL_HYBRID, ACO, PSO
    total_cost DOUBLE NOT NULL,
    compute_time DOUBLE NOT NULL,
    routes JSON NOT NULL,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP

```

## 迁移指南

### 已废弃的算法名称

以下算法名称已废弃,不再使用:

| 废弃名称 | 替换为 |
|---------|-------|
| `GA` | `genetic` |
| `ga_mdvrp_java` | `genetic` |
| `GA_MDVRP_JAVA` | `genetic` |
| `ga_rl_hybrid` | `GA_RL_HYBRID` |
| `hybrid` | `GA_RL_HYBRID` |
| `ant_colony` | `ACO` |
| `genetic_optimized` | `genetic` |
| `pso_optimized` | `PSO` |
| `aco_optimized` | `ACO` |

### 数据库迁移

如果数据库中存在旧的算法名称,需要执行以下SQL进行迁移:

```sql
-- 更新旧的算法名称
UPDATE solution SET algorithm = 'genetic' WHERE algorithm IN ('GA', 'ga_mdvrp_java', 'GA_MDVRP_JAVA', 'genetic_optimized');
UPDATE solution SET algorithm = 'GA_RL_HYBRID' WHERE algorithm IN ('ga_rl_hybrid', 'hybrid');
UPDATE solution SET algorithm = 'ACO' WHERE algorithm IN ('ant_colony', 'aco_optimized');
UPDATE solution SET algorithm = 'PSO' WHERE algorithm = 'pso_optimized';
```

## 注意事项

1. **大小写敏感**: 算法名称严格区分大小写,必须完全匹配
2. **前后端一致**: 前端发送的算法名称必须与后端和Flask服务保持一致
3. **数据库存储**: 数据库中存储的算法名称必须是标准名称之一
4. **错误处理**: 如果接收到不支持的算法名称,应返回明确的错误信息
