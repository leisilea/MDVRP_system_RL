# 重规划功能完整流程分析

## 📋 **功能概述**

重规划功能用于处理**道路阻塞场景**下的动态路径调整。当某些道路被阻塞时,系统会重新规划受影响车辆的路径,同时保持未受影响车辆的路径不变。

```javascript
const handleEdgeSelected = (edge) => {
  // edge = { from: 数据库ID, to: 数据库ID }
  
  // 检查是否已存在
  const exists = blockedEdges.value.some(
    e => (e.from === edge.from && e.to === edge.to) || 
         (e.from === edge.to && e.to === edge.from)
  )
  
  if (exists) {
    ElMessage.warning('该路段已被选择')
    return
  }
  
  // 转换为算法索引
  const fromIndex = convertToAlgorithmIndex(edge.from)
  const toIndex = convertToAlgorithmIndex(edge.to)
  
  // 存储双重ID: 数据库ID用于显示,算法索引用于发送
  blockedEdges.value.push({
    from: edge.from,           // 数据库ID (显示用)
    to: edge.to,               // 数据库ID (显示用)
    fromIndex: fromIndex,      // 算法索引 (发送用)
    toIndex: toIndex           // 算法索引 (发送用)
  })
  
  ElMessage.success(`已添加阻塞路段: ${edge.from} → ${edge.to}`)
}
```

}

### **7. AlgorithmService转发 (AlgorithmService.java)**

```java
public Map<String, Object> replan(Map<String, Object> request) {
    log.info("开始重规划，参数: {}", request);
    
    try {
        // 调用Python服务的 /api/replan 端点
        String url = algorithmServiceUrl + "/api/replan";
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<Map<String, Object>> httpRequest = new HttpEntity<>(request, headers);
        
        ResponseEntity<Map> response = restTemplate.postForEntity(
            url, httpRequest, Map.class
        );
        
        Map<String, Object> result = response.getBody();
        
        if (result != null && Boolean.TRUE.equals(result.get("success"))) {
            log.info("重规划完成");
            return (Map<String, Object>) result.get("data");
        } else {
            String error = result != null ? (String) result.get("error") : "未知错误";
            throw new RuntimeException("重规划失败: " + error);
        }
        
    } catch (Exception e) {
        log.error("重规划调用失败", e);
        throw new RuntimeException("重规划服务调用失败: " + e.getMessage(), e);
    }
}
```

**转发到**: `POST http://localhost:5000/api/replan`

### **8. Python Flask处理 (app.py)**

```python
@app.route('/api/replan', methods=['POST'])
@handle_exceptions
def replan_routes():
    start_time = time.time()
    data = request.json
    
    # 验证输入数据
    _validate_replan_request(data)
    
    # 解析请求参数
    depots = data['depots']
    customers = data['customers']
    routes = data['routes']
    blocked_edges = data['blocked_edges']
    vehicle_positions = data.get('vehicle_positions')
    algorithm = data.get('algorithm', 'GA')
    params = data.get('params')
    
    logger.info(f"收到重规划请求 - 仓库数: {len(depots)}, 客户数: {len(customers)}, "
               f"路径数: {len(routes)}, 阻塞路段数: {len(blocked_edges)}, 算法: {algorithm}")
    
    # 导入简化版重规划服务
    from replanning.service_simple import SimpleReplanningService
    from replanning.models import (
        DepotInput, CustomerInput, RouteInput, BlockedEdgeInput
    )
    
    # 转换输入数据为数据类
    depot_inputs = [DepotInput(**depot) for depot in depots]
    customer_inputs = [CustomerInput(**customer) for customer in customers]
    route_inputs = [RouteInput(**route) for route in routes]
    blocked_edge_inputs = [BlockedEdgeInput.from_dict(edge) for edge in blocked_edges]
    
    # 创建简化版重规划服务实例
    service = SimpleReplanningService()
    
    # 执行重规划
    result = service.replan(
        depots=depot_inputs,
        customers=customer_inputs,
        routes=route_inputs,
        blocked_edges=blocked_edge_inputs,
        vehicle_positions=vehicle_positions,
        algorithm=algorithm,
        params=params
    )
    
    # 转换响应为字典格式
    response_data = result.to_dict()
    
    logger.info(f"重规划完成 - 新路径数: {result.num_routes}, "
               f"成本变化: {result.cost_difference:.2f} ({result.cost_change_percent:.2f}%), "
               f"耗时: {result.solve_time:.3f}s")
    
    return jsonify({
        'success': True,
        'data': response_data,
        'timestamp': time.time()
    })
```

### **9. 重规划核心逻辑 (service_simple.py)**

重规划服务的核心步骤:

1. **识别受影响的路径**: 检查哪些路径包含阻塞路段
2. **提取未完成客户**: 从受影响路径中提取车辆位置之后的客户
3. **创建临时仓库**: 将车辆当前位置作为临时仓库
4. **重新求解**: 使用指定算法对未完成客户重新规划路径
5. **合并路径**: 将新路径与未受影响的路径合并
6. **计算成本变化**: 对比重规划前后的总成本

## 🔑 **关键概念**

### **1. 车辆位置 (Vehicle Positions)**
- **作用**: 标识每辆车当前在哪个客户点
- **格式**: `{ vehicleId: customerId }`
- **初始化**: 对话框打开时随机选择路径上的一个客户
- **用途**: 确定哪些客户已完成配送,哪些还需要重新规划

### **2. 阻塞路段 (Blocked Edges)**
- **作用**: 标识哪些道路被阻塞,不能通行
- **格式**: `{ from_node: 索引, to_node: 索引 }`
- **双重ID**: 数据库ID用于显示,算法索引用于计算

### **3. 临时仓库 (Temporary Depots)**
- **作用**: 将车辆当前位置作为新的起点
- **原因**: 车辆已经在路上,不能回到原仓库
- **属性**: 位置坐标、原车辆ID、可用车辆数(1辆)

### **4. 受影响路径 (Affected Routes)**
- **识别**: 检查路径是否包含阻塞路段
- **处理**: 只重新规划受影响的路径,保持其他路径不变

### **5. 未完成客户 (Unserved Customers)**
- **定义**: 车辆位置之后的客户
- **提取**: 从受影响路径中提取
- **重新分配**: 使用算法重新规划这些客户的配送

## 🎯 **重规划的核心价值**

1. **动态响应**: 实时处理道路阻塞等突发情况
2. **局部优化**: 只重新规划受影响的部分,保持其他路径不变
3. **成本可控**: 显示重规划前后的成本变化
4. **灵活性**: 支持多次重规划,可以继续添加新的阻塞路段
5. **可视化**: 在地图上直观显示阻塞路段、临时仓库和新路径
